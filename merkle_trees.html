Merkle Trees & Data Structures

The structure that lets a blockchain prove a single transaction is real — without making anyone download the whole chain to check it.

What it is

One hash, standing in for everything

A Merkle tree takes a pile of data — transactions, files, anything — and hashes it down into a single fixed-length fingerprint called the root. It does this in layers: every piece of data gets hashed individually, then pairs of those hashes get hashed together, then pairs of those get hashed together, and so on, until only one hash is left at the top.

The shape is just a binary tree. The substance is that every node above the leaves is nothing but the hash of its two children.

Why blockchains use it

Cheap proof, expensive forgery

A block header only needs to store the root — a few dozen bytes — no matter how many thousands of transactions are inside. Change a single transaction anywhere in the tree, and the hash above it changes, which changes the hash above that, all the way up to the root. The root can no longer match what everyone already agreed on, so tampering is caught instantly, without re-checking every transaction by hand.

Better still, you can prove one transaction belongs in a block by sharing just a handful of sibling hashes — a Merkle proof — instead of the entire block. The diagram below lets you try both of those things yourself.

    Where else this shows up

    Not just blockchains

    • Bitcoin hashes every block's transactions into a Merkle root stored in the block header, so light clients can verify a payment without downloading the full block.
    • Ethereum goes further with a Merkle Patricia trie — a Merkle tree merged with a lookup structure — to commit to the entire account and storage state, not just transactions.
    • Git is a Merkle DAG: every commit, tree, and file blob is content-addressed by its hash, which is why changing one line anywhere changes the commit hash.
    • IPFS splits files into chunks and links them the same way, so any piece of content can be verified independently by its hash.
